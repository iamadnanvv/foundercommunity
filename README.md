# FounderHuntCommunity

A premium community platform for SaaS founders, AI founders, indie hackers, and early-stage investors. Members get a curated feed, resource vault, startup showcase, member directory, and events — gated behind a ₹599 one-time lifetime payment.

---

## Tech Stack

| Layer       | Choice                                                              |
| ----------- | ------------------------------------------------------------------- |
| Framework   | [TanStack Start](https://tanstack.com/start) v1 (React 19 + Vite 7) |
| Styling     | Tailwind CSS v4 (`src/styles.css`) + shadcn/ui + Lucide icons       |
| Data layer  | TanStack Query (loader + `useSuspenseQuery`)                        |
| Backend     | Lovable Cloud (Supabase Postgres + Auth + Storage, RLS on)          |
| Server code | `createServerFn` (RPC) + `createFileRoute` server routes (webhooks) |
| Payments    | Razorpay (custom HMAC-verified checkout, no webhook)                |
| Runtime     | Cloudflare Workers (workerd) via Nitro                              |

### File naming conventions

- **Routes** use dot-segments: `users.$id.edit.tsx` → `/users/:id/edit`.
- **Server functions** live in `*.functions.ts` files (client-safe import path).
- **Server-only helpers** live in `*.server.ts` files (excluded from client bundle).
- **shadcn primitives** stay under `src/components/ui/` with their original names.

## Routing

- File-based via the TanStack Router Vite plugin → emits `src/routeTree.gen.ts`.
- `src/routes/_authenticated/route.tsx` is the auth gate: anything inside redirects to `/auth` when no Supabase session is present.
- Use `<Link to="/path">` and `useNavigate()` — never raw `<a href>` for internal nav.

---

## Auth Flow

1. User signs in at `/auth` via Google OAuth or email/password (`supabase.auth.signInWithPassword` / `signUp` / `signInWithOAuth`).
2. On sign-up, a Postgres trigger inserts a row into `public.profiles` and assigns the `founder` role in `public.user_roles`.
3. `_authenticated/route.tsx` subscribes to `supabase.auth.onAuthStateChange` and redirects unauthenticated users to `/auth`.
4. Server functions that need the user use `.middleware([requireSupabaseAuth])`, which:
   - Validates the `Authorization: Bearer <jwt>` header (attached automatically by `attachSupabaseAuth` middleware in `src/start.ts`).
   - Provides `{ supabase, userId, claims }` to the handler via `context`.

If you add a new protected server function, **do not call it from a public route loader** — SSR/prerender has no session and the build will fail with `Unauthorized`. Call from components via `useServerFn` + `useQuery`, or only from loaders under `_authenticated/`.

## Database

Tables (all in `public`, all RLS-on, all with explicit `GRANT`s):

| Table         | Purpose                                       |
| ------------- | --------------------------------------------- |
| `profiles`    | One row per user. `is_paid` gates premium UI. |
| `user_roles`  | `(user_id, role)` — never store role on `profiles`. |
| `posts`       | Community feed posts with `category`.         |
| `post_likes`  | Upvotes for posts.                            |
| `resources`   | Template/playbook library.                    |
| `startups`    | Member showcase + upvote count.               |
| `payments`    | Razorpay order/payment audit trail.           |
| `events`      | Community events.                             |

Helper: `public.has_role(_user_id uuid, _role app_role)` — `SECURITY DEFINER`, used inside RLS policies to avoid recursion.

### Adding a new table

Use the Supabase migration tool and always include in the same migration:

```sql
CREATE TABLE public.thing (...);
GRANT SELECT, INSERT, UPDATE, DELETE ON public.thing TO authenticated;
GRANT ALL ON public.thing TO service_role;
ALTER TABLE public.thing ENABLE ROW LEVEL SECURITY;
CREATE POLICY "owners read own" ON public.thing FOR SELECT TO authenticated
  USING (user_id = auth.uid());
```

Without `GRANT`s, the Data API returns a permission error regardless of policies.

## Razorpay Payments

Files: `src/lib/razorpay.functions.ts`, `src/routes/pricing.tsx`.

Flow (no webhook):

```
[client] /pricing
   │
   │ 1. createRazorpayOrder()  ─── createServerFn (requireSupabaseAuth)
   │                                creates Razorpay order via REST,
   │                                inserts payments row (status=pending)
   │
   │ 2. Razorpay checkout modal (checkout.razorpay.com/v1/checkout.js)
   │
   │ 3. verifyRazorpayPayment(orderId, paymentId, signature)
   │      HMAC-SHA256(orderId|paymentId, KEY_SECRET) === signature
   │      → payments.status = 'success'
   │      → profiles.is_paid = true (via admin client)
   │
   ▼
navigate('/dashboard')
```

To test without charging real money, swap the keys to `rzp_test_*` in Lovable Cloud secrets.

## Server Function Pattern

```ts
// src/lib/foo.functions.ts
import { createServerFn } from '@tanstack/react-start';
import { requireSupabaseAuth } from '@/integrations/supabase/auth-middleware';
import { z } from 'zod';

export const doThing = createServerFn({ method: 'POST' })
  .middleware([requireSupabaseAuth])
  .inputValidator((d) => z.object({ id: z.string() }).parse(d))
  .handler(async ({ data, context }) => {
    const { supabase, userId } = context;
    const { data: row, error } = await supabase
      .from('things').select('*').eq('id', data.id).single();
    if (error) throw new Error(error.message);
    return row;
  });
```

Call from a component:

```tsx
import { useServerFn } from '@tanstack/react-start';
import { useQuery } from '@tanstack/react-query';
import { doThing } from '@/lib/foo.functions';

const fn = useServerFn(doThing);
const { data } = useQuery({ queryKey: ['thing', id], queryFn: () => fn({ data: { id } }) });
```

## Design System

- Theme tokens live in `src/styles.css` as CSS variables (OKLCH) — dark + gold.
- Use semantic Tailwind classes (`bg-background`, `text-foreground`, `bg-primary`, `text-gold`, etc.).
- **Never hardcode colors** like `bg-black` or `text-[#fff]` — they bypass theming.
- Font: Bricolage Grotesque (loaded via `<link>` in `__root.tsx`).
- Icons: `lucide-react`.

## Common Pitfalls

| Symptom                                                  | Cause / Fix                                                                 |
| -------------------------------------------------------- | --------------------------------------------------------------------------- |
| `Unauthorized` during build                              | Protected server fn called from a public route loader. Move to component.   |
| `Invalid server function ID` in browser                  | Stale client bundle. Hard refresh.                                          |
| `permission denied for table X`                          | Missing `GRANT` on the table.                                               |
| `window is not defined` during SSR                       | Browser-only code at module scope. Move inside a client-only effect.        |
| `process.env.X is undefined`                             | Read it inside `.handler()`, not at module top-level.                       |
| Blank page after navigation but URL is right             | Parent route missing `<Outlet />`.                                          |
